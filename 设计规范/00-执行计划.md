## 阶段 1：响应式重构

**优先级**：🔴 P0
**目标文件**：`index_mobile.html`、`app_mobile_new.js`

### 任务 1.1 — 视口单位替换

全屏容器（如 3D Canvas 容器 `#container`）：`100vh` → `100dvh`
**参考**：通用规范 01 §2.1

### 任务 1.2 — Safe Area 适配

- Header（标题栏）：`padding-top: env(safe-area-inset-top, 0px)`
- 底部 Strip（`#strip`）：`padding-bottom: max(8px, env(safe-area-inset-bottom))`
- 底部 ActionBar（随机/保存按钮区）：`padding-bottom: env(safe-area-inset-bottom, 0px)`
**参考**：通用规范 01 §2.2

### 任务 1.3 — 字号图标用 clamp()/CSS 变量改写

**替换目标**（`index_mobile.html` 中全部硬编码尺寸）：
- 随机按钮、保存按钮、标题文字 → `var(--text-*)`
- 配色卡片尺寸 → `clamp()` 自适应
- 颜色条高度 → `var(--icon-*)`
**参考**：通用规范 01 §4、§5

### 任务 1.4 — 间距系统化

所有硬编码的 `padding`、`margin`、`gap`、`border-radius` → `var(--space-*)`、`var(--radius-*)`
**关键位置**：
- `#strip-cards` 容器间距
- 色彩卡片之间的间距
- Header 内边距
- 按钮圆角
**参考**：通用规范 01 §4、§5

### 任务 1.5 — 手势与滚动规范

- `#strip-cards`（横向滚动区域）：`touch-action: pan-x; overscroll-behavior-x: contain;`
- 颜色弹窗（纵向滚动）：`touch-action: pan-y; -webkit-overflow-scrolling: touch`
- 页面主体：`body { overflow: hidden; touch-action: manipulation; }`（阻止 double-tap 缩放但保留原生滚动）
- ⚠️ 不要用 `touch-action: none` — 会阻止 strip 卡片横向滑动（Bug 2 根因）
- canvas 元素自身设置 `touch-action: none` 来阻止三指手势干扰 3D 场景
**参考**：通用规范 01 §8、03 §6

## 阶段 2：UI 浮层优化

**优先级**：🔴 P0
**目标文件**：`index_mobile.html`、`app_mobile_new.js`

### 任务 2.1 — Header 透明化

- **位置**：`index_mobile.html` 中的标题栏区域
- **背景**：`background: transparent; border: none; box-shadow: none;`
- **层级**：`z-index: 100`
- **标题文字**：颜色 `var(--text-primary)`（深色，不使用 drop-shadow）
- **标题字号**：`var(--text-xl)`（`clamp(17px, 5vw, 22px)`，非 `--text-2xl`）
- **安全区**：`padding-top: env(safe-area-inset-top, 0px)`
- **布局**：flex，左右（左标题/右设置按钮）

### 任务 2.1a — Logo SVG 内联化 + CSS 变量控制颜色

**目标**：将 `<img src="Logo/logo_H.svg">` 和 `<img src="Logo/Brand_logo.svg">` 内联到 HTML，用 CSS 变量控制填充色，使深色模式能自动切换 logo 颜色。

**操作步骤**：
1. 复制 `Logo/logo_H.svg` 和 `Logo/Brand_logo.svg` 内容到 `index_mobile.html` 替代 `<img>` 标签
2. 将 SVG 内部的 `<style>` 中的 `fill` 值替换为 CSS 变量：
   - `logo_H.svg`: `cls-1 { fill: var(--logo-primary, #231f20); }` / `cls-2 { fill: var(--logo-accent, #c9926c); }`
   - `Brand_logo.svg`: `cls-1 { fill: var(--logo-primary, #231f20); }`
3. 在 `:root` 中定义浅色模式值，在 `[data-theme="dark"]` 中定义深色模式值
4. 删除 `Logo/` 目录下可能不再使用的 SVG 源文件引用

**CSS 变量定义**：
```css
:root {
  --logo-primary: #231f20;
  --logo-accent: #c9926c;
}
[data-theme="dark"] {
  --logo-primary: #f5f5f7;
  --logo-accent: #d4a574;
}
```

**参考**：`02-深色模式设计规范.md`（CSS 变量系统）、此计划阶段 0 的 CSS 变量骨架

### 任务 2.2 — Strip（底部卡片栏）透明化

## 阶段 5：动效实施

**优先级**：🟡 P1（核心）→ 🟢 P2（增强）
**目标文件**：`app_mobile_new.js`、`index_mobile.html`

### 任务 5.1 — 3D 颜色切换动画

当前的 `animateColorChange()`（行1436）已有 350ms ease 动画，主要优化方向：
- 检查是否所有颜色切换路径都调用了该函数（`applyColor()` / `applyFanColor()` / `randomizeColors()`）
- 对齐动画时长到规范标准 0.3s（当前为 350ms）
- 对齐缓动曲线到 `cubic-bezier(0.4, 0.0, 0.2, 1)`（当前为手动 ease 算法）
- 参考 `04-动效设计规范.md §3.1` 中的标准实现

### 任务 5.2 — UI 过渡动画

| 交互 | CSS 属性 | 时长 | 曲线 |

| 卡片选中 | `transform, border-color` | 0.2s | ease-in-out |

| 配色方案增删 | `transform: scale`, `opacity` | 0.25s | `ease` |
