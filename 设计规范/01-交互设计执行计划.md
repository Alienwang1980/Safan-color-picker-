## 阶段 0：审核现有交互，找出违规项

### 待检查项

在开始修改前，先基于通用交互规范检查当前 ColorPicker 的交互违规项：

1. **touch-action: none 设在整个 body 上** — `index_mobile.html:15` `body { touch-action: none }`
   - 违规原因：`touch-action: none` 应只设置在 canvas 上（`app_mobile_new.js:172`已经设置），body 设 none 会阻止所有原生滚动
   - 影响：`#strip-cards` 横向 scroll 可能被 body 的 `touch-action` 阻断了
   - 修复方案：将 `body { touch-action: none }` 改为 `body { touch-action: manipulation }`（统一值，参考通用规范）

2. **无返回按钮的全屏页** — `detail-container` 的返回按钮位置
   - 规范要求：全屏页左上角必须有返回按钮
   - 当前实现：`#detail-container` 中有 `#btn-detail-back`（通过 JS 动态渲染），需要确认其始终存在

3. **Sheet 弹窗的手势关闭** — 当前 sheet 只有"关闭"按钮
   - 规范要求：Bottom Sheet 应支持下滑关闭
   - 当前实现：没有下滑关闭手势，只有 `click` 关闭
   - 修复方案：给 `#modal-overlay` 添加下滑 dismiss 手势

4. **点击遮罩关闭** — 点击 overlay 背景
   - 当前：点击 `#modal-overlay` 的背景通过 `onClick` 关闭
   - 规范：✅ 已实现，但需要确认是否误触关闭（点击 sheet 内部不关闭）

5. **编辑模式是否屏蔽旋转/缩放** — 编辑模式下 3D 场景模糊
   - 规范要求：编辑模式下不可操作 3D 场景
   - ✅ 当前已实现（`#canvas-container.edit-mode-blur` + `edit-mode-overlay`）

6. **Toast 提示** — 当前创建新配色后有无反馈？
   - 需要确认 `createNewProject()` 中是否调用了 Toast

7. **空状态引导** — 首次使用时
   - 当前：有 `btn-add-project` + 空状态文案 ✅

8. **3D 场景提示"点击模型更改配色"是否遮挡** — z-index 2
   - 规范要求：提示不遮挡交互
   - ✅ 已设置 `pointer-events: none`

## 阶段 1：修复 Body touch-action

**目标：** 移除阻止原生滚动的错误 CSS，使 strip 卡片列表能正常横向滚动。

**修改文件：** `index_mobile.html`

```diff
- body {
-   touch-action: none;
- }
+ body {
+   touch-action: manipulation;
+ }
```

- `touch-action: manipulation` → 允许 iOS 双指缩放/旋转（由 canvas 自己的 `touch-action: none` 覆盖），同时不阻止原生 scroll
- canvas 的 `touch-action: none` 已在 `app_mobile_new.js:172` 设置

**验证：**
1. 打开页面 → 尝试横向滑动 strip 卡片列表
2. iOS Safari 应流畅惯性滚动
3. 3D 场景应保持正常旋转/缩放（canvas 上的 touch-action: none 覆盖 body 设置）

## 阶段 2：添加 Sheet 下滑关闭手势

**目标：** 遵循 Bottom Sheet 设计规范，支持下滑关闭弹窗。

**修改文件：** `app_mobile_new.js` 中的 `openPicker()`/`closePicker()` 方法

**实现方案：**
1. 给 `#modal-sheet`（或 `.modal-sheet`）添加 touch 事件监听
2. 监听 touchstart 记录起始 Y
3. touchmove 中如果 Y 位移 > 20px 且方向向下 → translateY 跟随手指
4. touchend 时如果位移 > 80px → 关闭 sheet；否则回弹到原位置
5. 同时保留"关闭"按钮

**关键代码位置：**
- `openPicker()` — 打开弹窗的入口（需绑定手势）
- `closePicker()` — 关闭弹窗的出口（下滑关闭时调用）
