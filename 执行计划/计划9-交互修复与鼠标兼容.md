## 改动 2：透明色赋予模型支持 td

**文件**：`app_mobile_new.js`

**当前**：`_setMeshColor()` 只设置颜色、roughness、metalness，完全忽略 `td` 字段。

**数据来源**：`colors.js` 中 `PLA Translucent` 组的颜色对象有 `td` 字段（5~9，值越大越不透明）。

### 修改点

#### 2a. `_setMeshColor(mesh, hex, td?)`

```js
_setMeshColor(mesh, hex, td) {
  const color = parseInt(hex.replace('#', ''), 16);
  mesh.traverse(c => {
    if (c.isMesh && c.material) {
      if (!c.material.isMeshStandardMaterial) {
        c.material = new THREE.MeshStandardMaterial({ color: 0xffffff });
      }
      c.material.color.setHex(color);
      c.material.roughness = 0.65;
      c.material.metalness = 0.0;
      c.material.emissive.set(0x000000);
      c.material.emissiveIntensity = 0;
      
      // 透明度处理
      if (td !== undefined && td !== null) {
        const opacity = td / 10;
        c.material.transparent = true;
        c.material.opacity = opacity;
        c.material._origOpacity = opacity; // 用于 restoreAll 正确恢复
      } else {
        c.material.transparent = false;
        c.material.opacity = 1.0;
        c.material._origOpacity = 1.0;
      }
    }
  });
}
```

## 改动 3：鼠标兼容 — 长按编辑模式

**文件**：`app_mobile_new.js` — `_initEditMode()` 方法（第 722-756 行）

**当前**：仅监听 `touchstart/touchmove/touchend`，鼠标无反应

**改为**：替换为 `pointerdown/pointermove/pointerup`（PointerEvent 同时兼容鼠标和触屏）

```js
_initEditMode() {
  const container = document.getElementById('strip-cards');
  if (!container) return;
  let pressTimer = null;
  let pressStartX = 0;
  let pressStartY = 0;

  container.addEventListener('pointerdown', (e) => {
    const card = e.target.closest('.strip-card');
    if (!card) return;
    pressStartX = e.clientX;
    pressStartY = e.clientY;
    pressTimer = setTimeout(() => {
      navigator.vibrate && navigator.vibrate(50);
      this.enterEditMode();
    }, 500);
  });

  container.addEventListener('pointermove', (e) => {
    const dx = Math.abs(e.clientX - pressStartX);
    const dy = Math.abs(e.clientY - pressStartY);
    if (dx > 10 || dy > 10) clearTimeout(pressTimer);
  });

  container.addEventListener('pointerup', () => {
    clearTimeout(pressTimer);
  });
}
```

⚠️ **必须完全替换**：不能同时保留 touch 监听。PointerEvent 在触屏上会先触发 `pointerdown`，然后浏览器可能还会触发 `touchstart`。如果两者同时存在，长按 500ms 后编辑模式会被触发两次。

### 潜在风险

| # | 风险 | 评估 | 应对 |
|:-:|------|------|------|
| 3.1 | **指针离开卡片时计时器未清理** | mouse 可以离开元素而不触发 pointerup | 加 `pointerleave` 清理计时器 |
| 3.2 | **鼠标 vs 触屏的 `e.clientX/Y` 兼容性** | PointerEvent 的 clientX/Y 在两种设备上行为一致 | ✅ 无需额外处理 |
| 3.3 | **触屏上的 pointerdown + touchstart 双重触发** | iOS Safari 中 pointer 和 touch 事件栈共存 | 完全移除 touch 监听，只用 pointer ✅ |
| 3.4 | **编辑模式下卡片 click 事件被正确处理** | 第 853 行已 `if (this.state.isEditMode) return;` | ✅ 防御到位 |
| **3.5** | **全部删除后 strip 折叠，编辑模式无法退出**（与改动 6 关联） | projects 为 0 时 strip collapsed，用户看不到卡片无法点击外部退出 | deleteProject 中 `if (projects.length === 0) exitEditMode()` |

## 改动 4：鼠标兼容 — 卡片拖拽滚动

**文件**：`app_mobile_new.js` — `initBottomStrip()` 中新增

**问题**：`.strip-cards` 使用 `overflow-x: auto`，鼠标用户无法像触屏一样拖拽滑动卡片列表。滚轮横向（Shift+滚轮）不够直观。

**方案**：在 `strip-cards` 上绑定 PointerEvent 实现鼠标拖拽滚动。

```js
// 初始化拖拽滚动
_initStripDragScroll() {
  const container = document.getElementById('strip-cards');
  if (!container) return;
  let isDragging = false;
  let startX = 0;
  let scrollLeft = 0;
  let dragDist = 0;

  container.addEventListener('pointerdown', (e) => {
    // 只在卡片区域触发
    const card = e.target.closest('.strip-card');
    if (!card) return;
    isDragging = true;
    startX = e.clientX;
    scrollLeft = container.scrollLeft;
    dragDist = 0;
    container.setPointerCapture(e.pointerId);
  });

  container.addEventListener('pointermove', (e) => {
    if (!isDragging) return;
    const x = e.clientX;
    const dist = x - startX;
    dragDist = Math.abs(dist);
    container.scrollLeft = scrollLeft - dist;
  });

  container.addEventListener('pointerup', (e) => {
    if (!isDragging) return;
    isDragging = false;
    // 如果拖拽距离很小（≤5px），视为点击，由卡片自身的 click 事件处理
    // 这里不需要阻止 click，因为 pointer 事件在 click 之前触发
    if (dragDist <= 5) {
      // 视为点击，交由卡片 click 处理
    }
  });
}
```

⚠️ 注意：`pointerdown` 和卡片自身的 `click` 事件的关系：

1. pointerdown → pointermove（拖动）→ pointerup
2. 浏览器判断是否触发 click：根据 pointerup 和 pointerdown 之间的距离和时间
3. 如果拖拽超过一定距离，浏览器**不会**触发 click 事件

所以拖拽滚动和卡片点击是天然互斥的：拖拽时不会触发 click，点击（无移动）时会触发 click。✅

### 潜在风险

| # | 风险 | 评估 | 应对 |
|:-:|------|------|------|
| 4.1 | **与卡片 click 事件冲突** | pointerdown 设置 isDragging=true，所有卡片上的 click 都可能被取消 | 只在拖拽距离 > 5px 时认为是 drag，≤5px 交给 click 处理。但浏览器本身已经做了这件事（click 只在无/小位移时触发） |
| 4.2 | **与长按编辑模式的 conflict** | pointerdown 既用于长按（改动 3）又用于拖拽滚动（改动 4） | 两者的目标容器相同（`#strip-cards`），pointerdown 事件是叠加的，互不冲突。但是长按是在卡片上 500ms，拖拽立即响应 |
| 4.3 | **触屏上滚动惯性丢失** | 拖拽滚动后，`overflow-x: auto` + `-webkit-overflow-scrolling: touch` 的惯性滚动可能被覆盖 | `pointerup` 后不做 `preventDefault`，让浏览器自行处理。iOS 的 momentum scrolling 由系统控制，不受 PointerEvent 影响 |
| **4.4** | **setPointerCapture 干扰其他 UI** | 指针移出 strip-cards 后，setPointerCapture 会将事件继续重定向到容器，导致 header 按钮等无法点击 | 移除 setPointerCapture，改用 pointerleave 在指针离开时取消拖动状态 ✅ |
