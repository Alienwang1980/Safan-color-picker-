## 代码整体现状

手机版当前代码（`app_mobile_new.js` + `index_mobile.html`）仍为旧版设计，主要问题：
- 配色库用旧版 `selectedProjectId` 逻辑，无焦点方案概念
- 无全屏详情页（`#detail-container` 不存在）
- Header 和 Strip 有白色半透明背景，非透明浮层
- 随机按钮在 toolbar 内，Save 按钮尚未取消
- 无 `createNewProject()` / `openDetailPage()` 等方法

执行顺序：
1. 先执行**计划A（代码审查修复 — 第一批）** — 独立 Bug 修复，与第 5 步/第 7 步不冲突
2. 然后执行第 5 步 + 第 7 步（可并行）+ 计划A 第二批（可并行）
3. 再继续后续步骤 + 计划A 第三批

## UI 布局（最终确认）

```

[3D Canvas - 全屏占满]                                  ← z-index: 1
[随机配色 按钮]                                         ← action-bar，z-index: 100
[Bottom Strip: [+] [卡1] [卡2] ...]                   ← 透明浮层，+ 固定左，z-index: 100
```
