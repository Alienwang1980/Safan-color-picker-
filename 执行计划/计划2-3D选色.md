### 修改 2 — typeMap 简化

**文件**：`app_mobile_new.js` line 556-562

**当前**：
```javascript
getColorsByType(type) {
  const typeMap = {
    'Basic': 'Basic',
    'matte': 'Matte',
    'translucent': 'Translucent',
  };
  const key = typeMap[type] || 'Basic';
  const groupKey = 'PLA ' + key;
  const group = this.colorGroups[groupKey];
  return group || [];
}
```

**改为**：
```javascript
getColorsByType(type) {
  return this.colorGroups[type] || [];
}
```

### 修改 3 — 配色弹窗色块加 name + code

**文件**：`app_mobile_new.js` line 509-524

**当前**：
```javascript
renderModalGrid() {
  const grid = document.getElementById('color-grid');
  if (!grid) return;
  const type = this.state.modalColorType;
  const colors = this.getColorsByType(type);
  grid.innerHTML = colors.map((c, i) =>
    `<div class="swatch${i === 0 ? ' selected' : ''}" data-index="${i}" style="background:${c.hex}" title="${c.name}"></div>`
  ).join('');
  // ...
}
```

**改为**：
```javascript
renderModalGrid() {
  const grid = document.getElementById('color-grid');
  if (!grid) return;
  const type = this.state.modalColorType;
  const colors = this.getColorsByType(type);
  grid.innerHTML = colors.map((c, i) =>
    `<div class="swatch${i === 0 ? ' selected' : ''}" data-index="${i}" data-hex="${c.hex}">
      <div class="swatch-color" style="background:${c.hex}"></div>
      <div class="swatch-info">
        <div class="swatch-name">${c.name}</div>
        <div class="swatch-code">${c.code || ''}</div>
      </div>
    </div>`
  ).join('');
  // ...
}
```

**CSS 配合**：`.swatch` 需要从纯色块改为 flex 容器，容纳色块和信息两行。需确认 HTML 里 `.swatch` 的样式是否需要调整。

## 依赖

- `this.state.modalColorType` — 状态变量，初始值需与 `openModal()` 统一
- `this.getColorsByType(type)` — 配色查询函数，Tab 直接传完整 key（如 'PLA Basic'）不再经 typeMap
- `this.renderModalGrid()` — 色块渲染函数，修改后需兼容现有 CSS
- `window.__COLOR_GROUPS` — 配色数据（'PLA Basic' / 'PLA Matte' / 'PLA Translucent'）
- Logo 文件存在（已在第 1 步确认）

## 风险点

- 修改 3 的 CSS 布局：`.swatch` 如果当前是固定宽高色块样式，改成 flex 容器可能破坏现有布局
- 修改 2 需要确认是否有其他地方直接调用 `getColorsByType('Basic')` 而不是完整 key
