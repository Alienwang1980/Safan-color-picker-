# 第 8 步：数据整合（PC/手机版共用数据）

> 所有功能步骤（第 1-7 步）完成后执行，部署前必须完成

---

## 步骤目标

**完成标志**：
1. 抽离 `__COLOR_GROUPS` + `__BATTERY_COLORS` 为独立文件 `colors.js`
2. `index_v2.html`（PC 版）和 `index_mobile.html`（手机版）都引用 `colors.js`
3. 两端使用同一份配色数据，不再各自内联拷贝

| 数据 | PC 版位置 | 手机版位置 | 整合后 |

| `__COLOR_GROUPS` | `index_v2.html` 内联 script | `index_mobile.html` 内联 script | → `colors.js` |
| `__BATTERY_COLORS` | `index_v2.html` 内联 script | `index_mobile.html` 内联 script | → `colors.js` |
| **3D 模型（GLB）** | `Models/` 目录 | `Models/` 目录 | **✅ 已共用，无需整合** |
| **Three.js 模块（lib/）** | `lib/` 目录 | `lib/` 目录 | **✅ 已共用，无需整合** |
| **Logo SVG** | `Logo/` 目录 | `Logo/` 目录 | **✅ 已共用，无需整合** |
| Three.js CDN | 共用同一 importmap | 共用同一 importmap | ✅ 已共用 |
| localStorage | `sidan_projects`（PC） | `sidan_projects_mobile`（手机） | ✅ 已隔离，各自独立 |

**结论**：部署时 `Models/`、`lib/`、`Logo/` 目录和两个 HTML 一起部署即可，模型、Three.js 模块和 Logo 天然共用。

## 执行内容

1. 从 `index_v2.html` 提取 `__COLOR_GROUPS` + `__BATTERY_COLORS` → 新建 `colors.js`
2. 修改 `index_v2.html`：删除内联数据，添加 `<script src="colors.js">`
3. 修改 `index_mobile.html`：删除内联数据，添加 `<script src="colors.js">`
4. 验证 PC 版和手机版功能不变
