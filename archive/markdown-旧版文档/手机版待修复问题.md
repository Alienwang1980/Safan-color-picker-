## 问题 3 — Strip 卡片选中逻辑（中严重度）

**文件**：`app_mobile_new.js` — `renderBottomStripProjects()`

**当前实现**：
```javascript
const isCenter = (activeProjectIndex === -1 && i === 0) || i === activeProjectIndex;
```

**问题**：`activeProjectIndex === -1`（无匹配）时，`i === 0` 被设为 center，但此时不应有任何 center

**修复**：当 `activeProjectIndex === -1` 时，不设 center
```javascript
const isCenter = activeProjectIndex !== -1 && i === activeProjectIndex;
```

## 问题 7 — getColorsByType typeMap 冗余/有误（低严重度）

**文件**：`app_mobile_new.js` line 556-562

**当前实现**：
```javascript
getColorsByType(type) {
  const typeMap = {
    'Basic': 'Basic',      // 但 Tab data-type 传进来的是 'PLA Basic'
    'matte': 'Matte',
    'translucent': 'Translucent',
  };
  const key = typeMap[type] || 'Basic';  // 'PLA Basic' 查不到 → fallback 到 'Basic'
  const groupKey = 'PLA ' + key;         // 拼完恰好是 'PLA Basic' — 结果对但逻辑绕
  const group = this.colorGroups[groupKey];
  return group || [];
}
```

**问题**：
- Tab `data-type` 已经是完整 key `'PLA Basic'`，不需要 `typeMap` 中间映射
- `'PLA Basic'` 查不到 typeMap，fallback 到 `'Basic'`，再拼前缀 —— 巧合正确，掩盖逻辑错误

**修复**：typeMap 改为直接返回
```javascript
getColorsByType(type) {
  return this.colorGroups[type] || [];  // type 已经是 'PLA Basic'
}
```
