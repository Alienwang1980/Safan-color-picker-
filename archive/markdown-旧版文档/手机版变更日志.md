### 跳过

- **计划 A — Reference Mode（参考模式配色）**
  - 原因：手机版用户场景与 PC 版不同。手机版用户直接在 3D 模型上看最终效果，不需要「色卡 flat 模式」作为参考。
  - 决定：手机版仅保留 Official（原色）模式，`applyColor()` 保持简单签名，不引入 `colorType` 参数。
  - 日期：2026-05-30

- **计划 C — 配色模式切换 UI**
  - 原因：依赖计划 A，且手机版仅原色模式，切换按钮无实际功能。
  - 日期：2026-05-30

## [已完成]

### 2026-05-30 — 计划 A/C 跳过（重大变更）

**变更来源**：用户确认手机版只需原色模式，不需要 Reference mode 色卡参考效果。

#### 计划 A 跳过的完整影响分析

| 项目 | 原计划 A | 跳过后的影响 |
|---|---|---|
| `applyColor(mesh, hex, colorType, extra)` | 重写为 4 参数签名 | ❌ 不执行，保持 `applyColor(partName, hex)` 简单签名 |
| Reference mode（参考模式） | 新增 roughness=1 / metalness=0 材质切换 | ❌ 不执行，手机版仅有原色模式 |
| `_reapplyAllColors()` | 切换模式时重新应用所有颜色 | ❌ 不执行，无模式切换 |
| `_applyReferenceLighting()` | Reference 模式专用低强度灯光 | ❌ 不执行 |
| `_applyStandardLighting()` | 恢复标准灯光 | ❌ 不执行 |
| `findColor()` | 从 COLOR_GROUPS 按 hex 查找颜色对象 | ❌ 不执行 |
| `state.colorMode` | 新增 'official' / 'reference' 状态 | ❌ 不执行 |
| `_reapplyAllColors() bugfix` | PC 版 translucent 死循环 bug 修复 | ❌ 不适用于手机版（已从记忆体删除） |

#### 计划 C 连带跳过

- 计划 C（配色模式切换 UI）依赖计划 A，**连带跳过**
- Toolbar 不再需要 `[原色] [参考模式]` 切换按钮
- Dark 主题移至第 3 阶段（执行计划 E）

#### 对其他计划的影响

| 计划 | 原依赖 | 新依赖 | 变化 |
|---|---|---|---|
| 计划 B（保存/加载） | 可与 A 并行 | 无 | ✅ 不变，`applyProjectColors()` 已是简单签名，无需修改 |
| 计划 D（Profile 快照） | 依赖 A | 无 | ✅ `restoreProfile()` 改用简单 `applyColor(mesh, hex)` 签名 |

#### 执行顺序变更

```
变更前：A → B（可并行） → C → D → E
变更后：B（第1阶段）→ D（第2阶段）→ E（第3阶段）
       └── 可并行 ──┘
```
